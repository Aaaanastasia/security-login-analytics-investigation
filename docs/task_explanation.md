# Task Explanation

## Context

This assessment evaluates analytical skills, critical thinking, anomaly detection, hypothesis generation, and communication using real-world-style data scenarios.

## Part A: Login Attempts Database

Review the login attempts dataset and identify at least five anomalies or suspicious login events.

For each case:

- Describe the observed behavior.
- Explain why it is a red flag.
- Include supporting visualizations.
- Show code or notebook logic used to reach the conclusion.

Additional discussion:

- What additional data fields would improve detection?
- What other suspicious authentication patterns should be monitored?

## Part B: Surge in New Accounts

A KPI dashboard shows a significant increase in new customer sign-ups over the past week.

Tasks:

- Identify at least five key factors to investigate.
- Design a dashboard to monitor and investigate this KPI.

## Deliverable Strategy

The submitted work is packaged as:

- A presentation for interview-style review.
- A runnable notebook/script for reproducibility.
- Documentation explaining the task, assumptions, data handling, and analytical decisions.

The presentation emphasizes not only what was found, but why specific analytical choices were made and why alternatives were rejected.
